# Interview Preparation Guide: Performance Engineering

Welcome! This is a **comprehensive, high-quality guide** to help you prepare for senior performance engineer interviews.

## Philosophy

**Deep explanations over quick summaries.** Every page:
- Starts from absolute basics
- Builds to senior-level reasoning
- Includes real code examples
- Explains the "why" not just the "what"

No stub files. No "coming soon." Only complete, usable guides.

## All Topics Covered

### Core Concepts
- **Java Memory Management** - From "what is memory?" to diagnosing memory leaks
- **Performance Testing Fundamentals** - How to design tests that answer real questions
- **Types of Performance Tests** - Load, capacity, stress, spike, soak, volume, scalability

### Testing Strategy
- **Software Testing Foundations** - Risk-based approach to quality
- **ISTQB for Senior Engineers** - Beyond certification memorization
- **Test Execution Plans** - From planning through results analysis

### System Optimization
- **Common Performance Problems** - CPU, memory, I/O, lock, and misconfiguration bottlenecks
- **Important KPIs** - Response time, throughput, error rate, utilization, business impact

### Architecture & Databases
- **Caching with Microservices** - Patterns, invalidation, stampede prevention
- **Microservices Performance** - Latency, parallelization, resilience patterns
- **Message Brokers** - RabbitMQ vs Kafka, guarantees, tradeoffs
- **Oracle Architecture** - How the database works inside
- **Query Optimization** - From slow to fast
- **RDBMS Optimization** - Connection pools, transactions, deadlocks, sharding

## How to Use This Guide

**For learning:**
1. Read a topic from start to end
2. Don't skip the explanations (they matter)
3. Try the code examples
4. Explain to someone else (best learning method)

**For interviews:**
1. Review your weak areas
2. Practice explaining concepts
3. Use examples from guide (real-world relevant)
4. Connect performance to business impact

**For on-the-job:**
1. Reference for troubleshooting
2. Mental models for design decisions
3. Verify your understanding matches guide

## Learning Path

If you have time for everything:
1. Java Memory Management (foundational)
2. Performance Testing Fundamentals
3. Common Performance Problems
4. Microservices Performance
5. All other topics

If you have limited time, prioritize by your background:
- **Backend developer?** Start with Java Memory + Database topics
- **QA/Tester?** Start with Performance Testing
- **Architect?** Start with Microservices + KPIs
- **DevOps?** Start with RDBMS Optimization + Monitoring

Each topic is self-contained, read in any order.
