# ISTQB (Senior Engineer View)

ISTQB is a **shared vocabulary** and **framework for structured testing**, not a certification checklist.

## The Testing Principles (How Seniors Apply Them)

- Testing shows **presence** of defects, not absence
- Exhaustive testing is impossible → **prioritize by risk**
- Early testing saves time and money → **shift-left**
- Defects cluster → **focus on hot spots**
- Testing is context dependent → **your strategy differs by system**
- Absence-of-errors fallacy → **passing tests ≠ right product**

## Test Levels and Types

**Levels:** unit, integration/service, system, acceptance

**Types:** functional, performance, security, reliability, usability

**Senior move:** Define quality attributes per service (SLOs, error budgets).

## The ISTQB Test Process as a Delivery Tool

1. **Planning:** scope, objectives, risks, entry/exit criteria
2. **Analysis:** derive test conditions from requirements + risks
3. **Design:** test cases and test data
4. **Implementation:** execute with controlled configuration
5. **Reporting:** translate metrics into decisions and risk posture

## How to Talk About ISTQB in Interviews

"It gives shared language and structured process. I use principles for risk-based strategy and exit criteria definition."
